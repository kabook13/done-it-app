# 🔄 תהליך עבודה - higayonbarie.co.il

## 📋 סקירה כללית

מדריך זה מסביר איך לעבוד עם הפרויקט בצורה מסודרת ויעילה.

---

## 🛠️ תהליך פיתוח

### 1. הכנת סביבת עבודה

```bash
# פתח את הפרויקט ב-Cursor
File → Open Folder → F:\cursor\files\higayonbarie-site
```

### 2. עבודה על פלאגין

**שלבים:**
1. פתח את הפלאגין ב-`plugins/your-plugin/`
2. ערוך את הקבצים
3. בדוק את השינויים
4. שמור את השינויים

**דוגמה:**
```
plugins/goodlife-wordle/
├── wordle.php          ← ערוך כאן
├── hebrew-words.json   ← עדכן מילים כאן
└── ...
```

### 3. עבודה על Theme

**שלבים:**
1. פתח את ה-Theme ב-`themes/hello-theme-child/`
2. ערוך קבצים (functions.php, style.css, וכו')
3. בדוק את השינויים
4. שמור את השינויים

**דוגמה:**
```
themes/hello-theme-child/
├── functions.php       ← הוסף פונקציות כאן
├── style.css          ← ערוך עיצוב כאן
└── js/                ← הוסף סקריפטים כאן
```

## 📝 כללי עבודה

### ✅ לפני התחלת עבודה

1. **בדוק מה צריך לעשות:**
   - קרא את המשימה/בקשה
   - הבן מה צריך לשנות
   - בדוק אם יש תיעוד רלוונטי

2. **צור גיבוי:**
   - אם עובדים על קובץ חשוב, שמור עותק
   - או צור branch ב-Git (אם משתמשים)

3. **תיעוד שינויים:**
   - כתוב הערות בקוד
   - עדכן תיעוד אם צריך

### ✅ במהלך העבודה

1. **עבוד בצורה מסודרת:**
   - שמור קבצים לעיתים קרובות
   - בדוק שגיאות (linter)
   - בדוק שהקוד עובד

2. **כתוב קוד נקי:**
   - השתמש בשמות משתנים ברורים
   - הוסף הערות במקומות מורכבים
   - עקוב אחרי סטנדרטים של WordPress

3. **בדוק שינויים:**
   - בדוק שהפונקציונליות עובדת
   - בדוק שלא שברת משהו קיים
   - בדוק תאימות לדפדפנים

### ✅ אחרי סיום עבודה

1. **בדוק סופית:**
   - הרץ בדיקות
   - בדוק שגיאות
   - בדוק שהכל עובד

2. **תעד שינויים:**
   - עדכן README אם צריך
   - כתוב הערות על מה שונה
   - עדכן CHANGELOG אם יש

3. **שמור:**
   - שמור את כל הקבצים
   - צור גיבוי אם צריך
   - העלה לשרת (אם מוכן)

## 🔄 תהליך העלאה לשרת

### שלב 1: הכנה

1. **בדוק שינויים:**
   ```bash
   # בדוק מה שונה
   git status  # אם משתמשים ב-Git
   ```

2. **צור גיבוי:**
   - גבה את הקבצים הקיימים בשרת
   - שמור גיבוי ב-`backup/`

### שלב 2: העלאה

1. **העלה קבצים:**
   - העלה רק את הקבצים ששונו
   - השתמש ב-FTP/SFTP או בסקריפט

2. **בדוק בשרת:**
   - בדוק שהאתר עובד
   - בדוק שהשינויים נראים
   - בדוק שאין שגיאות

### שלב 3: אימות

1. **בדוק פונקציונליות:**
   - בדוק שהכל עובד
   - בדוק בדפדפנים שונים
   - בדוק במכשירים שונים

2. **תעד:**
   - כתוב מה הועלה
   - שמור תאריך העלאה
   - עדכן תיעוד אם צריך
